# Provider State Example

This example demonstrates the full provider state system: simple states, parameterised states, and `fromProviderState()` for values generated by the provider during setup.

## What You'll Learn

- Simple provider states: `given('state name')`
- Parameterised states: `given('state {param} exists', { param: 'value' })`
- `fromProviderState('${expression}', exampleValue)` for provider-generated values
- State handler return values and generator expression substitution
- Selective provider verification using environment variables

## Running the Example

```bash
npm install
npm test
```

## How It Works

**Consumer** uses `fromProviderState('${accountId}', 42)` for the account's internal ID. The `42` is an example value for the mock; the provider will substitute a different (real) ID during verification. This is important for fields that the provider's database generates — the consumer can't know the real ID in advance, only that it will be an integer.

**Provider state handler** receives `{ accountNumber: 'ACC-001' }` from the parameterised state. It seeds the account and returns `{ accountId: <real id> }`. Pact uses this returned value to substitute into the response before matching it against the contract. The state handler return value is the bridge between `fromProviderState()` and real provider-generated data.

**Selective verification**: when debugging a large provider test suite, you can run a subset of interactions by setting environment variables before `npm run test:provider`:

```bash
PACT_DESCRIPTION="GET request for account" npm run test:provider
PACT_PROVIDER_STATE="an account with number" npm run test:provider
PACT_PROVIDER_NO_STATE=true npm run test:provider
```
